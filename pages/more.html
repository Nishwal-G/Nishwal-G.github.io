Open access website building

All of the files for this website are free to view and download via the GitHub link on this site. I’m also providing beginner-friendly templates and a step-by-step guide here so that anyone can build their own website with little to no prior experience.

Website building is often made to feel difficult, hidden behind paywalls, or locked behind tools that require subscriptions. That means many people are excluded from having a personal website through no fault of their own. My aim here is to make the basics clear, slow, and accessible — and to help you leave with a working site you can control yourself.

If you spot something confusing, inaccessible, or unclear, please get in touch — I’m happy to improve this page.

What you will build

You will build a small multi-page website hosted for free using GitHub Pages. Your site will have:

  • A homepage (index.html)
  • Extra pages (About, Projects, CV, More)
  • A shared stylesheet (.css) so the whole site has one consistent look
  • Optional JavaScript for nice extras (scroll progress bar, fade-in effects)

Key ideas (in simple terms)

If you’ve never built a site before, here are the only concepts you need at the start:

  • HTML files (ending in .html) are the content of your site: the words, headings, links, and page structure.
  • CSS files (ending in .css) are the style: colours, fonts, spacing, layouts, and how things look.
  • Folders keep things organised (for example: images in assets/).
  • GitHub Pages is a free service that turns your files into a public website.

You can think of it like this: HTML = the building, CSS = the paint and interior design.

How your website will be structured

When you’re done, your GitHub repository (your online folder) will look roughly like this: 

your-website/
  index.html
  pages/
    about.html
    projects.html
    cv.html
    more.html
  stylesheets/
    base.css
  js/
    main.js
  assets/
    images-and-files-go-here.png
    cv.pdf

The most important file is index.html because it is the “front door”. GitHub Pages looks for index.html first when someone visits your site.

Step-by-step: publish your first website (beginner version)

  1. Create a GitHub account
    Go to GitHub and sign up. Choose a username you are happy to show publicly.
  2. Create a new repository
    On GitHub, click the + in the top-right → New repository.
  3. Name the repository
    You have two common options:
    • Option A (recommended for beginners): name it anything (e.g. my-website) and then enable GitHub Pages in Settings.
    • Option B (classic personal site): name it yourusername.github.io. This creates a website directly at that address.
  4. Make the repository public
    Choose Public so GitHub Pages can host it for free.
  5. Create your first file: index.html
    Inside the repository:
    • Click Add file → Create new file
    • Name it exactly: index.html
    • Paste in a template (provided below)
    • Scroll down and click Commit changes
  6. Create folders (directories) on GitHub in your browser
    GitHub doesn’t have a “New Folder” button in the simple view — instead you create a folder by typing it into the filename.
    • Click Add file → Create new file
    • In the filename box, type something like: pages/about.html
    • GitHub will create the folder (pages) automatically
    • Paste template content, then commit
  7. Create your stylesheet folder
    Repeat the same trick:
    • Create stylesheets/base.css and paste the CSS template
    • Commit changes
  8. Create your JavaScript file (optional)
    Repeat again:
    • Create js/main.js and paste the JS template
    • Commit changes
  9. Enable GitHub Pages
    In your repository:
    • Click Settings
    • Find Pages in the sidebar
    • Under Build and deployment, choose: Deploy from a branch
    • Select your branch (usually main) and the folder (/ (root))
    • Click Save
    After this, GitHub will show you the link to your website.
  10. Visit your website
    It may take a minute or two to appear. Once it’s live, refresh the page and you should see your homepage.

How to edit your site (the simplest workflow)

  1. Click the file you want to edit in your repository (e.g. index.html).
  2. Click the pencil icon (Edit).
  3. Change the text (or links, headings, etc.).
  4. Commit changes at the bottom.
  5. Refresh your website to see the update.

If your update doesn’t appear immediately, wait a moment and refresh again — GitHub Pages can take a short time to redeploy.

Custom domain (yourname.com instead of github.io)

You can keep the free GitHub address, or you can buy a custom domain like yourname.com. Buying a domain is optional — your website will work either way.

Step 1: buy a domain

You can buy domains from many providers (for example: Namecheap, Google Domains alternatives, GoDaddy, Cloudflare Registrar, etc.). Choose a provider you trust and search for the domain you want.

Step 2: tell GitHub Pages your domain

  1. Go to your repository Settings → Pages
  2. Find Custom domain
  3. Type your domain (e.g. yourname.com) and save

Step 3: update DNS at your domain provider

DNS is the system that points your domain name to GitHub’s servers. Your domain provider will have a “DNS” or “Manage DNS” page.

GitHub Pages typically uses:

  • A records for the root domain (yourname.com)
  • CNAME for www.yourname.com pointing to your GitHub Pages address

After updating DNS, it can take some time to “propagate” (spread across the internet). GitHub Pages will show when your domain is correctly connected.
